What is a Promise in JavaScript and How to Create Promises?

To create a Promise, we have to use a constructor function. This constructor function (is to create different instance of particular type) takes a function as an argument and this function is called "Executor function". This Executor function takes two parameters as arguments, one is resolve and another is reject. These resolve and reject both are functions given by JavaScript.

Inside the Executor function we have to call either resolve or reject function. But in some different scenarios and logic we can call both resolve and reject.

The Executor function that handles the Asynchronous operation. 

        new Promise((resolve, reject) => {})

        //Executor function 
        (resolve, reject) => {
          //logic goes here
        }
State of Promise:

1 - pending: The promise is in the pending state when the Executor function starts the execution.
2 - fulfilled: when promise is resolved the state of the promise is fulfilled.
3 - rejected: when the promise is rejected the state of the promise is rejected.

Result of Promise:

1 - undefined: when the state of the promise is pending the result of promise is undefined.
2 - value: when promise resolve(value) is called.
3 - error: when promise reject(error) is called.

Handling Promises

There are three handler methods to handle promise

      .then()
      .catch()
      .finally()

Promises Chaining

  • Rule 1:

    Every promise gives a .then() handler. Every resolved promised gives only .then() handler, it doesn't gives .catch() handler. In case of every rejected promise it gives .catch() handler, it doesn't gives .then() handler method

  • RUle 2:

    There are three tasks we can accomplish within a .then() method.
    1. We can return another promise for a async operation.
    2. We can return any value for a synchronous .
    3. We can throw an error.

  • Rule 3:

    An error can be rethrown from the .catch() handler to handle the error later. In this case, the control will go to the next closest .catch() handler

  • Rule 4:

    Unlike .then() and .catch() .finally() handler doesn't process the result value or error. It just passes teh result as it is to the next handler

  • Rule 5:

    Calling a .than() multiple times on a single promise is not a chain

Handle Multiple Promises

There are four utility methods to handle promises:

  • Promise.all():

    if any of the promise fails to resolve, Promise.all() method returns reject. that means all the promises should be fulfilled of resolved

  • Promise.any():

    if any of the promise is fulfilled or resolved among promises given in an array then it returns that promise immediately as a value but not array unlike Promise.all()

  • Promise.allSettled():

    this method waits for the all promises to be settled(resolved or rejected). whether it is fulfilled or whether it is rejected it gives the output of everything. in case of fulfilled it gives the status and value object and in case of rejection it gives the reason and says why it is rejected

  • Promise.race():

    this method returns the fastest and quickest result which has been fulfilled of resolved
