✅ “this” in Classes

The keyword "this" refers to the object not the class itself. This is the reason we can create different properties for different instances.

      class Car {
        constructor(name) {
          this.name = name
        }
        printThis() {
          console.log(this);
        }
      }

      const car = new Car("BMW");
      console.log(car); // Car {name: 'BMW'}
      car.printThis() // Car {name: 'BMW'}

✅ Class as Expression

Classic Expression:

      class AClass {
        constructor() {}
      }
      const a = new AClass();

Variable Class:

      const Employee = class {
        constructor() {}
      }
      const emp = new Employee();

Named Class:

      const Dept = class Department {
        constructor() {}
      }
      const dept = new Dept();

✅ Getter and Setter

Getter and Setter are the special methods that is used to modify or validate the property of a method

      class Animal {
        constructor(name) {
          this.name = name;
        }

        printName() {
          console.log(`The original property of the object is: ${this.name}`); //The original property of the object is: Get the animal name Tiger
        }

        get name () {
          if(!this._name) {
            console.warn(`No valid animal name found!`);
            return;
          }
          return `Get the animal name ${this._name}`;
        }

        set name (value) {
          if(!value || value.length < 2) {
            console.warn(`The name property cannot be empty or less than 2 characters!`);
            return;
          }
          if(value === "Cat") 
          this._name = "Tiger";
        }
      }

      const animal = new Animal("Cat");
      animal.printName();
      console.log(animal.name); // Get the animal name Tiger

✅ Static Method

In the class static method can be created using the keyword "static" before a method name. A static method can be define with the arguments and this method can be accessed on the class name itself and any parameter can be provided and this parameter is not depended on class properties. The this inside a static method refers to class itself. 

      class MyClass {
        static staticMethod() {
          console.log(this); // this returns the hole class itself
        }
      }
      MyClass.staticMethod();

      class User {
        constructor(name, email) {
          this.name = name;
          this.email = email
        }

        greet() {
          console.log(`Hi I am ${this.name}`);
        }

        static isValidEmail(email) {
          return email.includes("@") && email.includes(".");
        }

        static createGuest() {
          return new User("Guest", "guest@emal.com")
        }

      }

      const user = new User("Shiplu", "shiplu@email.com");
      user.greet(); // Hi I am Shiplu

      const isValidEmail = User.isValidEmail("test@test.com");
      console.log(isValidEmail); // true
      
      const guestUser = User.createGuest(); 
      console.log(guestUser); // User {name: 'Guest', email: 'guest@emal.com'}
      guestUser.greet(); // Hi I am Guest

✅ Private/Public

Public: The fields and methods are accessible from the outside of the class that is called Public fields or methods

Private: The fields and methods are only accessible in the class is called Private fields or methods.
