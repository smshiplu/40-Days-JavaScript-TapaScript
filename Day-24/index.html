Why async/await?

async/await comes into picture to avoid some difficulties of promise. In case of promise we need to use it with .then() .catch() and it increase the difficulties of codes While reviewing or reading. It can not be easy to understand the reasons of using .then(), .cath() or finally(). that's why the async and await is very handy while handling promise

The async Keyword

The async keyword is used to create a asynchronous function. While defining the function the async keyword is got set in front of of function. 

      //classic function
      async function() {}
      
      // variable function
      const promise = async () => {}

      // anonymous function
      async () => {}

The async function always returns a promise whether we return any value or explicitly returns a promise.

      async function foo() {
        return 101;
      }
      console.log(foo()); //Output: PromiseÂ {: 101}

      async function bar() {

The await Keyword

The await keyword is used to handle the promise which is returned from the async function. This is the alternate of .then() method. But the whenever we use await keyword it mut be wrapped inside another async function. It cannot be use at global level. Await keyword suspends the function execution, until the promise settled (resolve or reject)

      async function bar() {
        return Promise.resolve(101);
      }
      async function tacklePromise() {
        const retBar = await bar();
        console.log(retBar)
      }
